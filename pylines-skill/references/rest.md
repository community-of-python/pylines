# REST API Guide

Источник: community-of-python/pylines/rest.md

## Философия

REST — стиль, не стандарт. Мы не имплементируем HATEOAS, используем code-first подход (FastAPI/Litestar → Swagger UI).

## Основные правила

- Формат: **JSON**
- Протокол: **HTTP/2** или **HTTP/3**
- Каждый адрес — ресурс/сущность (существительное в URL)
- Глаголы — это HTTP методы (GET, POST, PUT, PATCH, DELETE)

## CRUD

```
GET    /rest/example/books/      — список
GET    /rest/example/books/1/    — одна сущность
POST   /rest/example/books/      — создать
PUT    /rest/example/books/1/    — обновить
DELETE /rest/example/books/      — удалить
```

Не все действия обязательны. Иметь только GET — нормально.

## URL conventions

### Схема
```
/rest/[имя-сервиса]/[имя-сущности]/     — REST
/rpc/[имя-сервиса]/[действие-глагол]/   — RPC
```

### Префиксы
- `/rest/*`, `/rpc/*` — базовое API (админка, фронт)
- `/rest-internal/*`, `/rpc-internal/*` — между микросервисами
- `/rest-public/*`, `/rpc-public/*` — внешние потребители

### Правила
- Разделитель: **дефис** (не подчёркивание) — чтобы отличать от Python-сущностей
- Слеши в конце закрывай
- **Вложенность запрещена**: максимум `/[тип]/[сервис]/[сущность]/`
- Не `/api/` — используй `/rest/` или `/rpc/`
- Единообразие числа: выбери единственное или множественное и придерживайся

### Вместо вложенного REST
```
# ❌ Запутанно
/rest/service/books/123/readers/12/books/2/

# ✅ GET-параметры
GET /rest/example/books/?user-id=5
```

## Версионирование

- Не делай без чёткой необходимости (поддерживать версии дорого)
- Нужно, если есть неопределённый круг клиентов (мобильные приложения)
- Версию размещай в **Accept заголовке** (мета-информация запроса)
- Пакет: [fast-version](https://github.com/community-of-python/fast-version)

## Обратная совместимость

Новые поля — необязательные, со значениями по умолчанию.
